Promise 是什么？

先想象一下你在一家港式甜品店点餐，点餐的流程通常是这样的： 选品 -> 支付 -> 等待 -> 取餐 -> 用餐 -> 走人

现在思考一下等待 -> 取餐这一步骤，我们如何知道什么时候该去取餐了呢？答案就是当你点完餐之后服务员会给你一个通知器（通常是一个小圆盘），当我们的餐做好之后，这个通知器会亮起，你看到这个就知道：哦，我的餐做好了，该去取了。

在这个例子中，这个通知器就好像我们的 Promise，甜品店作为生产者，我们作为消费者，这个通知器就是 Promise。生产者完成生产后通过通知器通知消费者。

// 生产者
function dessertShop(resolve, reject){
	setTimeout(() => {
		resolve('dount')
	}, 2000)
}

let notifier = new Promise(dessertShop)

// .then(...)就是消费者
notifier.then((dessert) => {
	if(dessert === 'dount'){
		console.log('get!')
	}
})
  • resolve(value)表示任务完成，结果为 value
  • reject(error)表示任务失败，原因为 error

由new promise实例化的对象notifier具有以下内部属性：

  • state：pending，resolve之后变成fulfilled，如果是reject那么就变为rejected
  • result：最初是undefined，resolve之后是value，如果是reject那么就是error

这样想象，做餐这个过程就是pending，那么此时的result自然是undefined，当餐做好了（resolve）状态变为fulfilled，我们就得到这个餐品，即value，如果因为原材料不足等原因导致无法出餐（reject），那么状态就是rejected，我们得到了一个error（店家总得向我们解释为什么无法出餐）。

Promise 基本语法

let promise = new Promise(fn)，实例化一个promise对象，参数fn是一个函数表示生产者，fn接受两个参数resolve和reject，用于通知promise实例化对象结果。

promise.then(resolvedHanlder, rejectedHandler)，.then就是消费者，接受两个参数，参数为函数，第一个参数表示针对fulfilled结果作处理，第二个参数表示针对rejected作处理。
